[![P]][M] Рабочий процесс глазами менеджера
===========================================

  - в треккере создается задача.  
  - программист включается в работу.  
  - работает в своей отдельной ветке:  
    - правит исходники.  
    - правит документацию.  
    - правит историю.  
  - затем сливает ветку с мастером.  
  - фиксирует очередной релиз.
